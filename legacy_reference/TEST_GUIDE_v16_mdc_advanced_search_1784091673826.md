# KDRG 관계 검색기 v16 테스트 가이드

## 테스트 목적

이번 버전의 핵심은 두 가지다.

1. `MDC 04`, `호흡기계`처럼 MDC 자체를 검색하고 소속 ADRG를 탐색할 수 있는지
2. 여러 코드를 입력했을 때 단순 ADRG 교집합이 아니라 **같은 조건식 안의 연결인지, 서로 다른 OR 조건식에 분산된 관계인지**를 구분하는지

관계검색 결과는 최종 질병군 판정이 아니다. 화면에서 이 제한이 명확하게 보이는지도 함께 확인한다.

---

## 1. MDC 검색 테스트

### TC-MDC-01 · MDC 04 정식명
- 검색유형: 전체
- 검색어: `MDC 04`
- 기대결과:
  - 왼쪽에 `MDC 04 · 호흡기계` 1건이 우선 표시
  - 상세에 ADRG 57개, AADRG 수, A/B/C 분포 표시
  - 전체/A군/B군/C군 탭 표시

### TC-MDC-02 · MDC 04 별칭
- 검색어: `MDC04`, `호흡기계`
- 기대결과: TC-MDC-01과 동일한 MDC 04 상세

### TC-MDC-03 · MDC 05
- 검색어: `MDC 05`, `MDC05`, `순환기계`
- 기대결과:
  - MDC 05 상세
  - ADRG 104개 표시

### TC-MDC-04 · 숫자 단독 충돌 방지
- 검색유형: 전체
- 검색어: `04`
- 확인점:
  - 숫자만으로 MDC 04를 강제 해석하지 않는지
  - 코드 검색 결과와 MDC 검색이 불필요하게 섞이지 않는지

### TC-MDC-05 · MDC→ADRG→뒤로가기
- `MDC 05` 검색
- 목록에서 `F600` ADRG 상세 진입
- 뒤로가기 클릭
- 기대결과:
  - MDC 05 목록으로 복귀
  - 기존 스크롤 위치가 유지

## 2. 복수 코드 관계검색 핵심 테스트

### TC-REL-01 · 서로 다른 OR 조건식에 분산되는 사례
- 검색 1: 수술·처치코드 `O1311`
- 조건: AND
- 검색 2: 수술·처치코드 `O1326`
- 기대결과:
  - `E011` 표시
  - 상태: **서로 다른 OR 조건식에 분산**
  - O1311은 조건식 1의 시술명 table1
  - O1326은 조건식 2의 시술명 table2
  - 두 코드를 하나의 조합조건으로 해석하면 안 된다는 경고

이 사례가 가장 중요한 테스트다. 단순 ADRG 교집합과 공식 조건식 구조를 구분하는지 확인한다.

### TC-REL-02 · 같은 조건식·같은 TABLE의 두 코드
- 검색 1: 수술·처치코드 `O1326`
- AND
- 검색 2: 수술·처치코드 `O1351`
- 기대결과:
  - `E011`, `E018` 표시 가능
  - 같은 조건식 내 공통 연결
  - E011에서는 `ADC02 미포함`이 추가 확인사항으로 표시
  - 같은 TABLE에 코드 2개가 있어도 공식 규칙이 두 코드를 모두 요구한다는 뜻은 아니라는 안내

### TC-REL-03 · F136 OR 조건 중 동일 조건식
- 검색 1: 수술·처치코드 `M6552`
- AND
- 검색 2: 수술·처치코드 `M6554`
- 기대결과:
  - `F136` 표시
  - 상태: 같은 조건식 내 공통 연결
  - 조건식 2 또는 조건식 1에서 table2/table3 연결이 명확히 보임

### TC-REL-04 · 최소 2개 시술 요구사항
- 검색 1: 상병코드 `E1050`
- 검색 2: 수술·처치코드 `M6597`
- 검색 3: 수술·처치코드 `M6605`
- 조건: AND
- 기대결과:
  - `F194`와 `F195`가 함께 나타날 수 있음
  - F194: `시술명 table2 중 2개 이상` 구조상 입력코드 2개 일치 표시
  - F195: 같은 TABLE에 두 코드가 포함되지만 두 코드를 모두 요구한다는 뜻은 아니라는 경고
  - 검색기가 F194로 최종 판정하지 않아야 함

### TC-REL-05 · 주진단과 기타진단 역할 구분
- 검색 1: 상병코드 `I210`
- 검색 2: 기타진단코드 `I110`
- 조건: AND
- 기대결과:
  - `F600` 표시
  - 조건식 2에서 주진단 table2 + 기타진단 table3 연결
  - 두 코드 유형이 서로 바뀌어 표시되지 않음

### TC-REL-06 · 시간조건 때문에 하나로 판정할 수 없는 사례
- 검색 1: 수술·처치코드 `M5850`
- 검색 2: 수술·처치코드 `O7020`
- 조건: AND
- 기대결과:
  - `E501`, `E511` 모두 표시
  - 각각 인공호흡 96시간 이상/미만 추가조건 표시
  - 코드만으로 E501 또는 E511 한쪽을 선택하지 않음

### TC-REL-07 · 미포함 조건의 안전성
- 검색 1: 수술·처치코드 `O1326`
- 검색 2: 부가코드 `ADC02`
- 조건: AND
- 기대결과:
  - `E018`은 표시 가능
  - `E011`은 양의 조합 결과로 표시되지 않음
  - E011의 `ADC02 미포함` 조건을 ADC02 포함으로 오인하지 않음

### TC-REL-08 · 잘못된 코드유형
- 검색 1 유형: 상병코드
- 코드: `O1311`
- 기대결과:
  - 현재 데이터에서 해당 유형으로 찾지 못했다는 안내
  - 관계검색을 실행하지 않음

### TC-REL-09 · 중복 코드 입력
- 동일 코드와 동일 유형을 두 줄에 반복
- 기대결과:
  - 중복 입력 경고
  - `2개 이상` 조건을 동일 코드 중복으로 충족한 것처럼 처리하지 않음

### TC-REL-10 · 관계검색→ADRG 상세→뒤로가기
- TC-REL-01 실행
- E011의 전체 ADRG 상세 진입
- 뒤로가기 클릭
- 기대결과:
  - 복수 코드 관계분석 화면으로 복귀
  - 입력조건과 결과 목록 유지

## 3. 기존 기능 회귀테스트

- `I460` 검색 → F600/F651/F652 관계
- `ADC02` 단일 검색 → E018/E019만 양의 포함관계
- `F051` 검색 → F0511/F0512 복수 AADRG
- `F194` 검색 → 2개 이상/ADC04 OR UI
- `E034` 검색 → 233개 대형 TABLE 요약 및 펼치기
- `F800` 검색 → 대형 주진단 TABLE 검색·스크롤
- A/B/C 배지 색상 유지
- 부제목이 없는 ADRG에 빈 `()`가 나타나지 않음

## 4. 중점적으로 볼 문제

1. **거짓 양성**: 같은 ADRG에만 연결됐다는 이유로 서로 다른 OR 조건을 하나의 조합으로 표시하지 않는가
2. **거짓 확정**: `관련 ADRG`를 `조건 충족`이나 최종 DRG로 표현하지 않는가
3. **코드 역할**: 주진단/기타진단/수술·처치/부가코드가 정확히 구분되는가
4. **남은 조건**: 미입력 TABLE, 시간, 연령, 미포함 조건이 빠지지 않는가
5. **동일 TABLE 다중코드**: 여러 코드를 입력했다고 자동으로 여러 개 필요 조건으로 해석하지 않는가
6. **2개 이상 조건**: 서로 다른 코드 2개 입력 시에만 구조상 일치로 표시되는가
7. **탐색 복귀**: MDC 또는 관계검색 상세에서 ADRG로 이동한 뒤 원래 화면으로 돌아오는가
8. **성능과 가독성**: 결과가 많을 때 왼쪽 목록과 오른쪽 상세가 버벅이거나 지나치게 길어지지 않는가
