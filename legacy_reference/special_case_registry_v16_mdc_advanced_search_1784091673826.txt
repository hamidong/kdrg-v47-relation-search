[01] F136 · OR 조건식
- 사례 ID: SC_F136_OR_GROUPS
- 요약: 서로 다른 2개 조건식이 OR로 연결됨
- 검증 포인트:
  · 조건식 1/조건식 2 카드 분리
  · 카드 사이에 또는 구분 표시
  · 각 table 코드요약 및 펼치기 유지

[02] F194 · 최소개수·부가코드 복합조건
- 사례 ID: SC_F194_MIN_COUNT_OR_ADD_CODE

- 검증 포인트:
  · 2개 이상 요구사항 배지
  · OR 조건식 분리
  · ADC04 검색 연결
  · 추천/충족 판정으로 오인되지 않는 표현

[03] F213 · 신설코드·관련코드 주석
- 사례 ID: SC_F213_NEW_CODE_FOOTNOTE
- 요약: NCF06·NCF08 신설코드와 ICD-9-CM 관련코드 주석이 존재함
- 검증 포인트:
  · NCF06/NCF08를 실제 table 멤버로 검색
  · ICD-9-CM 3775/3789/3776은 현재 table 멤버가 아닌 관련 주석으로 구분
  · 향후 alias/search_index 확장 후보로 관리

[04] F220 · 장문·다중페이지 단일 TABLE
- 사례 ID: SC_F220_LONG_MULTI_PAGE_TABLE
- 요약: 하나의 시술명 TABLE이 PDF 2개 페이지에 걸쳐 있고 코드 40개를 포함함
- 검증 포인트:
  · 코드요약이 과도하게 화면 높이를 차지하지 않는지 확인
  · 상세 펼치기에서 40개 코드가 원문 순서대로 표시되는지 확인
  · 첫 페이지와 다음 페이지의 코드가 누락 없이 하나의 TABLE로 합쳐졌는지 확인
  · B군·일반진료질병군 색상 배지가 일관되게 표시되는지 확인

[05] F023 · 복수 조건식 OR
