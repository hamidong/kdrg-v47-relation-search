## 2. 우선순위 원칙 (충돌 시)

1. 분류집 본문(1) > 교정표(2) > 시술목록(3) > 질병군명칭(4) > 부록(5)
2. A/B/C 질병군 분류는 오직 별표1 PDF(6)만 근거로 삼는다 — 분류집 본문/부록/시술목록에서 A/B/C를 추정하지 않는다.
3. TABLE 소속(어떤 코드가 어떤 table에 포함되는지)은 분류집 본문(1)과 교정표(2)만 근거로 삼는다 — 시술목록(3)이나 부록(5)에서 TABLE 소속을 유도하지 않는다.

## 3. legacy_reference 격리

`legacy_reference/`에는 이전 v16(KDRG V4.6 세대) 스크립트·데이터·문서만 보관합니다.
이 폴더의 데이터는 코드/TABLE/ADRG 소스로 사용하지 않으며, UI·상호작용 구조를 참고하는
용도로만 남겨둡니다. 자세한 사용 제한은 `legacy_reference/NOTE.md`를 참고하세요.

## 4. 이번 1차 작업 범위와 남은 작업

이번 1차 작업은 기존 KDRG V4.7 파일럿(9개 ADRG, `kdrg_v47_pilot_special_cases_v1_FULL_*.zip`)의
UI·데이터를 새 프로젝트 구조로 옮기고, A/B/C 분류만 별표1 PDF로 재검증한 범위입니다.
분류집 본문(1,282쪽)·부록(776쪽)·교정표(HWPX)·시술목록(2,791행)·질병군명칭(2,700행) xlsx의
전체 파싱과 데이터 확장은 이번 범위에 포함하지 않았습니다(자세한 내용은 프로젝트
README.md의 "다음 단계" 참고).
